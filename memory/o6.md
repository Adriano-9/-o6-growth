## 2026-07-06 · O6 Bootstrap Pack (docs/bootstrap/)

### Contexto
Constituição operacional formal pedida pelo usuário (papel "Principal Software Architect", 6 pilares, protocolo auditar→plano→implementar→documentar→commit). Pedido inicial de 250-300 páginas de documentação.

### Decisão
Entregamos densidade real por tópico (22 arquivos, ~10.5k palavras) em vez de padding genérico para bater contagem de página — inflar volume documentando sistemas 🔴 planejados (Dark Factory, Higgsfield pipeline, Content Factory) como se fossem arquitetura real violaria a própria regra "nunca documente como implementado algo que ainda não existe". Cada arquivo declara status honesto 🟢🟡🔴.

### Trade-off
Volume menor que pedido literalmente, mas zero conteúdo inventado. `docs/bootstrap/README.md` documenta essa decisão explicitamente para não parecer que o pedido foi ignorado silenciosamente.

### Achado durante a auditoria
Confirmado: Codex nunca foi executado neste repositório (todo frontend foi construído por Claude Code). Higgsfield e 21st.dev nunca tiveram chamada de API real — só uso manual via connector do Claude Chat (Higgsfield, uma vez) ou aplicação manual de padrões conhecidos (21st.dev, nunca autorizado). "Factory Dark" não tem definição em nenhum lugar do histórico do projeto — documentado como charter em aberto, não inventado.

### Path tocado
`docs/bootstrap/` (22 arquivos novos)

## 2026-06-03 · Seed inicial do OS

### Stack travada
- **Next.js 16** (App Router + Turbopack) — breaking vs. training data. Todo agente novo lê `node_modules/next/dist/docs/` antes de tocar routing/data.
- **React 19** · **TypeScript 5 strict** · **Tailwind 4** (`@theme` block em `app/globals.css`)
- **Supabase JS** via singleton lazy em `app/offer-book/_lib/supabase.ts` — única fonte de verdade pra dados de cliente.
- Deps extras: `framer-motion`, `lucide-react`. Nada mais.

### Supabase
- Projeto **O6 growth** · id `wphrwidjokimfjfvyaym` · region `us-west-2` · `ACTIVE_HEALTHY`.
- Env em `.env.local`: `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY` (publishable key `sb_publishable_…`).
- RLS habilitada em todas as tabelas, **policy `anon all`** intencional enquanto não houver auth UI. Apertar quando entrar login.
- Migrations versionadas `NNN_descricao_snake` via `mcp__supabase__apply_migration`. Nunca DDL via `execute_sql`.

### Verticais prioritários
Clínicas · Estética · Odontologia · Fisioterapia · Advogados. Default mental: dono de clínica de estética com ticket médio R$ 2.500. Toda copy/placeholder reflete esse universo.

### Design system (tokens em `app/globals.css`)
- `brand-orange #FF5722`, `brand-cyan #00E5FF`, `brand-graphite #121212`, `brand-offwhite #F8F9FA`.
- Superfícies dark do OS: `bg-zinc-950` (canvas), `bg-zinc-900/40` (cards glass), `border-white/10`, `bg-white/[0.04]` (hover).
- Tier colors: `emerald-400` (≥70), `amber-300` (40-69), `red-400/500` (<40).
- Tipografia: Geist Sans Black 900 uppercase `tracking-tight` pra titles; labels micro `text-[10px] uppercase tracking-[0.18em]`.
- 6 pilares: Dark · SaaS Premium · Bento Grid · Glassmorphism · Mobile First · Conversão.

### Padrões compartilhados
- Path alias `@/*` → `./` (configurado em `tsconfig.json`).
- Diretórios privados com prefixo `_` (`_lib/`, `_components/`) — Next.js ignora pra routing.
- Mappers camelCase ↔ snake_case isolados em `_lib/store.tsx` por módulo. Quando uma tabela for compartilhada, mover pra `_lib/db.ts` global.

### Não-objetivos explícitos
- **Sem auth UI** (RLS anon all é decisão consciente).
- **Sem IA na lógica** — scores e plano de ação são 100% determinísticos. Política revisada quando ROI de IA estiver mapeado.
- **Sem lib de PDF** (jsPDF, react-pdf, puppeteer). `/print` usa `window.print()` + `@media print` CSS.
- **Sem state manager global** (Redux/Zustand/Jotai). Cada módulo controla state local; cross-module só via Supabase.
- **Sem CSS-in-JS**. Tailwind utility + `globals.css` only.

### Sub-rules culturais
- Bias para ação. Auto Mode é o default; o usuário redireciona quando precisa.
- `AskUserQuestion` só pra decisão que só o usuário toma. Nunca pra "posso prosseguir?".
- Verificação obrigatória pós-mudança: screenshot via Preview MCP (UI) + `execute_sql` (data).
- TaskCreate quando trabalho cruza ≥ 3 passos distintos.

### Trade-offs aceitos
- RLS permissivo = qualquer um com a anon key lê/escreve tudo. OK porque a URL/key não está pública e o uso é interno por enquanto.
- Mappers escritos à mão em vez de gerar tipos via `generate_typescript_types` — economia agora, dívida pequena. Gerar quando ≥ 6 tabelas.
- `window.print()` em vez de PDF lib = depende do browser do usuário renderizar bem. OK porque o cliente recebe o PDF final pelo navegador dele.
